# ---- STEP 9: GENERALIZED FUNCTION ----
def multi_hypothesis_bayes(hypotheses_dict, evidence_name="Evidence"):
    """
    General N-hypothesis Bayes calculator.
    Input: dict of {name: {"prior": float, "likelihood": float}}
    Returns: dict with posteriors added
    """
    # Validate priors
    total_p = sum(h["prior"] for h in hypotheses_dict.values())
    if abs(total_p - 1.0) > 1e-6:
        raise ValueError(f"Priors sum to {total_p}, not 1.0")

    # Compute numerators
    for h in hypotheses_dict.values():
        h["numerator"] = h["prior"] * h["likelihood"]

    # Denominator
    denom = sum(h["numerator"] for h in hypotheses_dict.values())

    # Posteriors
    for h in hypotheses_dict.values():
        h["posterior"] = h["numerator"] / denom

    return hypotheses_dict, denom

COMMON PITFALLS AND FAILURE MODES â€” SPECIFIC TO MULTI-HYPOTHESIS

PITFALL 1: MISSING HYPOTHESIS
    â†’ You forget to include one possible cause
    â†’ Denominator is TOO SMALL
    â†’ All posteriors are INFLATED
    â†’ Example: If we forgot Electrical Short:
        Denom = 0.270 + 0.190 = 0.460 (wrong, should be 0.660)
        P(Motor|OH) = 0.270/0.460 = 0.587 (wrong, should be 0.409)

PITFALL 2: OVERLAPPING HYPOTHESES
    â†’ "Motor Failure" and "Old Motor" are not mutually exclusive
    â†’ Counting the same cause twice inflates its posterior
    â†’ FIX: Merge overlapping categories

PITFALL 3: ZERO LIKELIHOOD
    â†’ If P(Evidence|Háµ¢) = 0 for some hypothesis
    â†’ That hypothesis gets posterior = 0 (completely eliminated)
    â†’ This is CORRECT behavior â€” but make sure the zero is justified
    â†’ A single wrong zero permanently kills a hypothesis

PITFALL 4: UNIFORM LIKELIHOOD
    â†’ If ALL hypotheses have the SAME likelihood
    â†’ Then posteriors = priors (evidence tells you NOTHING)
    â†’ This is also correct â€” uninformative evidence doesn't change beliefs
