COMMON PITFALLS AND FAILURE MODES

PITFALL 1: "OTHER" CATEGORY WITH WRONG LIKELIHOOD PROFILE
    â†’ "Other" is a catch-all for unknown diseases
    â†’ Its symptom profile is hard to define
    â†’ If set too high â†’ Other absorbs too much posterior mass
    â†’ If set too low â†’ Other gets zero posterior even when no disease fits
    â†’ BEST PRACTICE: Set "Other" likelihoods to population base rates
      of each symptom (marginal probabilities)

PITFALL 2: MISSING THE CLOSE RACE
    â†’ Flu=53%, COVID=42% looks like "Flu is the answer"
    â†’ But this is a VERY CLOSE race â€” nearly a coin flip
    â†’ Report the FULL distribution, not just the winner
    â†’ A 53/42 split demands FURTHER INVESTIGATION, not a conclusion

PITFALL 3: SYMPTOM PRESENCE BIAS
    â†’ Doctors record PRESENT symptoms more reliably than ABSENT ones
    â†’ "Rash: No" might mean "didn't check" rather than "confirmed absent"
    â†’ Treating unrecorded symptoms as ABSENT introduces bias
    â†’ Better: treat unrecorded symptoms as MISSING (omit from product)
    â†’ Naive Bayes handles this gracefully (just skip that feature)

PITFALL 4: TEMPORAL SYMPTOMS
    â†’ Fever on Day 1 vs Fever on Day 5 have different meanings
    â†’ Naive Bayes treats them the same
    â†’ Solution: include "days since onset" as a separate feature
    â†’ Or use time-series models (beyond basic Bayes)

PITFALL 5: PRIOR CONTAMINATION
    â†’ Doctor "knows" it's probably Flu (gut feeling)
    â†’ Sets P(Flu) = 0.80 based on intuition
    â†’ Bayes confirms Flu â†’ doctor feels validated
    â†’ But the prior was SUBJECTIVE and biased
    â†’ FIX: Use POPULATION-LEVEL data for priors, not individual intuition

# SECTION 9: GENERALIZED NÃ—M NAIVE BAYES FUNCTION
# ============================================================

def diagnose(disease_priors, symptom_likelihood_table, observed_symptoms,
             cost_table=None, verbose=False):
    """
    General N-hypothesis Ã— M-feature Naive Bayes diagnosis.

    Args:
        disease_priors: {disease: prior_probability}
        symptom_likelihood_table: {symptom: {disease: P(symptom=Yes|disease)}}
        observed_symptoms: {symptom: True/False/None}
            None = missing/unrecorded (will be skipped)
        cost_table: {disease: cost_of_missing} or None
        verbose: print details

    Returns:
        posteriors: {disease: posterior_probability}
        risk_scores: {disease: expected_risk} or None
    """
    # Validate priors
    total_prior = sum(disease_priors.values())
    assert abs(total_prior - 1.0) < 1e-6, f"Priors sum to {total_prior}"

    # Compute joint likelihoods
    joint_lk = {}
    for disease in disease_priors:
        log_joint = 0.0
        for symptom, is_present in observed_symptoms.items():
            if is_present is None:
                continue  # Skip missing features
            p_yes = symptom_likelihood_table[symptom][disease]
            p_obs = p_yes if is_present else (1 - p_yes)

            # Prevent log(0) â€” implicit Laplace-like floor
            p_obs = max(p_obs, 1e-10)
            log_joint += math.log(p_obs)

        joint_lk[disease] = math.exp(log_joint)

    # Numerators
    numerators = {d: disease_priors[d] * joint_lk[d] for d in disease_priors}

    # Denominator
    denominator = sum(numerators.values())

    # Posteriors
    posteriors = {d: n / denominator for d, n in numerators.items()}

    # Risk scores
    risk_scores = None
    if cost_table:
        risk_scores = {d: posteriors[d] * cost_table[d] for d in disease_priors}

    return posteriors, risk_scores
